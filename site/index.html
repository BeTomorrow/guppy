Mathematics input widget for the web






LaTeX Text ASCIIMath JSON XML


Human-friendly

  • WYSIWYG, backslashes optional.
  • Keyboard only; optional mouse interface.
  • Mobile keyboard.
  • LaTeX or plain text output.
  • Supports matrices and custom symbols.

Computer-friendly

  • Expressions stored in semantic XML.
  • XML or JSON output.
  • Generate and evaluate syntax trees.

Developer-friendly

  • Configure options globally or per editor.
  • Flexible API. Scriptable.
  • Style using CSS
  • MIT license

  • Can I evaluate expressions numerically?

    Yes! See the calculator and graphing examples.

  • Can I compare an input to a given expression?

    Yes, using the math-expressions library. Check out the quiz example.

  • Can I render just XML expressions on a page without an editor?

    Yes! Check out the article example.

  • How does it work?

    Data is stored internally in a custom XML format. Editing operations correspond loosely to XPath queries, and rendering to various output formats is a simple recursive transform (formerly XSLT). Display is done with KaTeX (with some modifications). See the behind the scenes guides.
